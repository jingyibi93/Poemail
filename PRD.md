# PRD 产品需求文档 (Product Requirement Document)

## 📌 1. 项目基本信息 (Basic Information)

*   **项目名称 (Project Name)**: Remix: 每日诗歌信箱 (Poetry Mailbox)
*   **产品定义 (Product Definition)**:  
    让英语学习变成最漫不经心、最轻柔的一个动作。每天打开信箱，拆阅一封送至心底的英文诗信与精选新词。通过极致的手画线稿（Hand-drawn line art）、怀旧机械打字机敲击音效、四段式沉浸式折纸火漆封口收藏交互与慢速语音低吟，创造一场纯粹、治愈、无压力的读诗英语轻餐。
*   **产品定位**: 慢节奏治愈系单人沉浸式英语多模态感官读诗工具。
*   **目标用户**:
    *   英语及诗歌文学爱好者；
    *   追求极致视觉美学、交互动效、声音氛围感反馈的用户群（如手作、慢节奏生活爱好者）；
    *   希望在碎片时间给心灵寻找一个“着陆角落”，无压力学习单词和品味文学的都市白领及学生。

## 🎨 2. 核心设计理念与美学原语 (Aesthetic & Design Philosophy)

### 2.1 极致手作感：不完美的、抖动的线稿美学 (Craftsmanship over Defaults)
*   **边框原语**: 彻底去除标准直板和圆滑的数字化矩形，全站以多款手绘圆珠笔风格的多点抖动框线 (`sketch-border-v1` ~ `v6` 等) 构筑所有卡片与按钮边缘。
*   **纸张纹理**: 背景层使用低对比度、泛黄并附有一层微小纤维肌理的牛皮板纸纹路 (`.paper-texture`, `.card-paper`) 作为卡片和页面的底层皮肤，摒弃一切现代高斯模糊与大平涂渐变。
*   **自然错落**: 所有按钮在静止状态和悬停时都有细微的非对称旋转（例如 `-1.8deg`、`2.2deg`），模拟手绘排版的自然感与凌乱美。
*   **留白涂鸦**: 卡片或信格缝隙、顶部标题处，零星分布几款手写笑脸、王冠等手绘矢量 Doodle，增添俏皮灵动。

### 2.2 绝对静音的低噪交互 (Whisper-quiet UI)
*   **零气泡弹窗**: 移除所有可能打扰用户专注度的黑色悬浮弹窗提示（Toast完全移除，仅保留极度含蓄的文字形态变换）。
*   **去数字绑架**: 界面中不设“连续打卡天数”、“等级积分”、“排行榜”或“金币充值”，极力降低用户的社交资本消耗与打卡焦虑。
*   **专注读诗**: 只有新词、音标、诗句和极其克制的发音按键，将干扰降低为零。

## ⚙️ 3. 核心功能模块详细需求 (Core Functional Requirements)

本应用由四个核心体验阶段（Phase）构成：

```
[阶段 1: 信箱格墙] 📬 ──(手触抽信)──> [阶段 2: 机械打字机] ⌨️ ──(慢速写透)──> [阶段 3: 诗笺卡片展示] 📄 ──(折叠火漆动效)──> [阶段 4: 珍藏柜阁] 🪵
```

### 3.1 核心一：心情信箱格墙 (Mailbox Wall Grid)
*   **功能描述**: 它是应用的主页。一堵由 6 个手绘信格组成的邮箱格墙，每个信格代表不同的心情主题。用户触碰即可抽出并自动“卷入”打字机。
*   **界面布局**:
    *   采用精美的 `2 columns x 3 rows` 规则网络排布，包含：
        *   **01 | Soft Landing (温柔着陆 - 对应疲累状态 `tired`)**: 以温柔、降噪的解压诗作为主；
        *   **02 | Quiet Room (安静角落 - 对应寂静独处 `quiet`)**: 精选内敛深邃、慢节奏的语句；
        *   **03 | Rain Note (下雨日记 - 对应温润细腻 `rainy`)**: 精选浪漫、略带一点潮湿水汽情绪的诗句；
        *   **11 | Today's Letter (今日来信 - 主信匣 `today`)**: 精显重点。右上角拥有一个俏皮上下跳动的红色 "For You ✉" 提示标，以及一页故意留白 peeking 明显突出、并在微风下自转角度抖动的物理信件舌头。
        *   **04 | Far Away (去往远方 - 对应逃离渴望 `escape`)**: 精选关于旷野、海岸线、流散和风的诗章；
        *   **05 | Little Glow (微光闪烁 - 对应暖意欢欣 `happy`)**: 囊括了朝阳、露水、微甜生活的诗笺。
    *   **顶部面板 (Daily Calendar Watermark)**: 居中优雅排布当天的完整英文日历水印 (例如 `JANUARY 12, 2026`) 以及手写风中文农历或星期 (例如 `• 1月12日 · 周一 •`)。
*   **交互逻辑**:
    *   Hover/点击任意卡片，卡片以手写微弧边缘进行 `1.03x` 缓慢微缩和阴影深度转移；
    *   点击时由于进入高多模态音效态，系统利用最纯净的浏览器事件拦截和唤醒 `AudioContext` 链，防止声音被宿主 iframe 限制挂起。

### 3.2 核心二：复古机械打字机体验 (Retro Typewriter Machine)
*   **功能描述**: 当用户抽出一封信后，跳转进入此页面。打字机将以一种有温度的击键速度，将英文诗文逐字敲到信笺上，让阅读产生渐进的感官满足。
*   **视听细节要求**:
    *   **动态音效池系统 (Dynamic Audio Rotating Pool)**:
        *   由于声音是此体验的核心灵魂，为防止大字快节奏下单音效重叠导致“吃音”或严重的“失真爆音”，系统建立了一组包含 14 个 `HTMLAudioElement` 的并发高速循环音轨备用池（针对普通字母敲击 `key.wav`，空格阻尼 `space.wav`，换轨物理拉杆 `return.wav`，以及行末撞针叮铃声 `bell.wav`）。
        *   自动加载失败时，立即启用 Web Audio API 极其纯净的无盘程序化模拟打击（使用低通和脉冲噪音叠加生成击针机械音），保证声音 100% 播放成功。
    *   **溅射涟漪敲击波 (Kinetic Waveforms)**:
        *   在每下打印头敲击纸张的瞬间，以击发字符的物理坐标（X、Y）为中心，喷涌并在 0.6 秒内淡出多层具有手绘笔触质感的圆形“涟漪声浪环”，将听觉转译成可触摸的浪漫视觉动效。
    *   **跳过逻辑 (Direct Rolling Out)**:
        *   虽然自动打印带来了仪式感，但考虑到信息效率，顶部和底部提供了一个十分克制的“直接取纸” (Skip) 操作，方便用户跳过长句。完成或跳过时，页面执行纸张拉出折叠声，并过渡到核心诗笺页。

### 3.3 核心三：诗歌信笺展示 (Poem Display View)
*   **功能描述**: 卡片展开的核心。一页写有精选新词、音标发音、例句和精美英文诗篇的双语卡片。
*   **界面特征**:
    *   **手绘氛围 Doodles 元素**:
        *   卡片内部不着痕迹地融入了两处精美浮现、不干涉文字的极细手绘 Doodle：右上角一盏热气腾腾的“手绘咖啡杯”，左下角是一支静静安放并在瓶身带有细密竖向阴影的“花瓶线条画”，营造诗意的午后面包屋角落。
    *   **今日落脚单词卡片**:
        *   加粗显示一个具有异域或者宁静韵味的“新词小憩” (`targetWord`)，配齐精细的英美音标发音（如 `[dɪˈlɪbərət]`）；
        *   提供完整的词性、细致的中文白话释义及一段贴切、不艰涩的精选诗化英文例句及流畅的中文对应释义。
    *   **诗句文字排布**:
        *   诗句必须采用经典的高对比度加重衬线体（Serif-en pairings），中文字体则为优雅精简的宋体/仿宋美学排印以拉开中英排版层次，多行分段，并贴心地靠中居左缩进。
*   **特色人机交互**:
    *   **极舒缓低吟诵 (Whisper Synthesis Reader)**:
        *   点击声音喇叭或在用户进入详情页 550ms 后（略迟于翻页切页的动画），应用借助浏览器内置 TTS（文本转语音），指定语速降至极静极缓的 `rate = 0.72` 倍速（约合每分钟读 80 词），专选 Google Natural 或 Aria等温柔声线慢读，不提供任何锐利刺耳的断句。
    *   **火漆折叠四段收藏交互 (4-Step Immersive Collection Animation)**:
        *   此功能是全案最高潮的交互细节。当用户决定“留存”这封信时，点击“加盖印信收藏”，界面触发一个长达 5.5 秒的多级折纸物理解压表演：
            1. **Step 1: 纸张滑入 (Slide Into Envelope)**: 完整的信纸左右颤抖，以折纸缩略角度慢慢“呲溜溜”滑入荷兰手绘信封卡槽内（伴随纸张拖拽声 `playEnvelopeInSound`）；
            2. **Step 2: 翻盖折回 (Fold Flap Down)**: 信封上方的小梯形翻盖，在慢速中翻转并咔嚓一声严丝合缝紧贴封底（伴随 `playPaperFoldSound` 厚质纤维折叠微音）；
            3. **Step 3: 火漆烙印 (Crimson Wax Stamp)**: 画面弹跳出一记复古血红色火漆蜡块并带着微小的弹性震颤（Spring Physics Animation）盖在信封翻盖接口中，形成一枚极佳的 Wax stamp 水印印油；
            4. **Step 4: 飞归箱阁 (Archiving to Keepsake Box)**: 完封的红白信封开始缩放，滑落空中并划过一道极轻的美学抛物线坠入到下方悄然打开、厚质手画的老木质首饰首饰箱里。最后伴随着木盖压紧、铜锁扣合（`playKeepsakeBoxSound`）的空谷沉声完全闭合，迸发精致的几颗手绘闪光气泡。
    *   **过客下一页 (Draw Another / Pass Card)**:
        *   用户在阅读完后可以通过一键挥舞“过客”，使卡片折叠推出，并在叮叮当当的打字声中随机刷新更换一首同一情绪类型的下一封诗。

### 3.4 核心四：收藏阁/珍藏柜 (KeepSafe Cabinet)
*   **功能描述**: 存放温暖记忆回忆的书架，用户从信箱搜罗而来的精美诗笺全部整装陈列于此。
*   **数据设计 (LocalStorage Backend)**:
    *   通过 `poetry_mailbox_collections` 字符序列数组安全持久地写入客户端 `localStorage`，不依赖用户注册和云端服务器，即便离线或处于断网环境中，收藏夹仍随时可用。
*   **界面原语**:
    *   展现出一方质朴温和的手画格九宫格古木屉橱；
    *   每个抽屉中静默放置着一封已经经过“折叠、扣盖、印泥火漆”的微型火漆信封，封套上醒目写着这封信当时对应的系统卡片 ID (例如 `No. 004`) 及来源情绪标签；
    *   点击任意封，页面会播放一记令人愉悦的拆封回弹缓冲，带用户秒速无缝重返诗歌信卡详情，进行重复性背诵和朗读学习；
    *   内附清脆的“取出盒外”（一键解锁归档）物理移除键。

## 🛠️ 4. 非功能性需求 (Non-Functional Requirements)

### 4.1 兼容性与 iframe 沙盒免挂运行 
*   **Web Audio API 通用解锁**: 必须满足多设备网页（及各类宿主容器 iframe 底座）由于静音政策拦截 Media 的限制。应用初始化时将事件全面托管在最外侧 window 轻击流及手势触碰中，通过对 `AudioContext` 静默载入与强力 resume，杜绝用户交互时发出报错或静音。
*   **TTS 备用声线回退**: Web Speech 语音引擎无法载入本地离线声包时，应无缝支持在主流美式女声、美式男声或系统首推英语人声（如 `Samantha`, `Aria`）之间渐进回退，防止读音断路。

### 4.2 性能与资源加载控制
*   **资源预加载 (Preloader Loop)**: 
    由于该产品属于极度侧重微互动、短音素并发的软硬件高度体验网，针对打字机的多个短 WAV 剪片，应该使用 Promise 合流，在主卡启动的首秒，以零开销并发在浏览器引擎内核中触发 fetch 并生成本地 Buffer 块。一旦用户按下信件，立刻无延迟播放，不引发任何由加载卡顿导致的声音画不同步缺陷。

### 4.3 响应式设计与视觉定宽 (Fluid Box Constraints)
*   **手机定级物理舞台 (Phone Stage)**:
    在超宽大屏浏览器或平板下，全应用被温柔锁合约束在一个宽约 `410px`、高约 `840px` 且带有优雅圆角与深色纯手工黑质外边框的虚拟手持容器（`Phone Stage`）内部，从而确保所有极细画作在线条颤动和缩放中永远不会因为比例被大范围拉伸而崩塌变形；而在小屏移动端时，其自动转回百分百无缝全屏平铺。

## 📈 5. 后续迭代规制 (Future Roadmap)

*   **V2.0 | 自选书写 (Handwritten Feed)**: 允许用户自己充当打字机，使用应用的高拟真键盘自己敲出想阅读的英文长句，并通过内置情感词网自动匹配适合的心情抽屉和线画 Doodles 背景。
*   **V2.5 | 诗信寄明信片 (Letter to Postcard Export)**: 支持将诗笺转化为一帧带手绘木框边缘、打字机戳、火漆印的日式极简美学卡片图，支持一键保存到本地进行分享。
